# zhc-od-resource

该仓库用于归档与发布多个独立的 HTML 页面。这些页面不针对特定域名或网站，而是作为通用资源进行管理。

## 项目简介

本仓库的主要目的是集中存放、版本管理和公开访问一组静态 HTML 页面。每个页面可以视为独立的功能单元或展示内容，适用于文档、工具界面、演示页面或信息展示等场景。

## 目录说明

仓库根目录下按页面进行组织，常见结构如下：

- `pages/`：存放主要的 HTML 页面文件
- `assets/`：页面引用的静态资源，如 CSS、JS、图片等
- `docs/`：页面相关的说明文档或示例
- `archive/`：不再更新但保留的旧版本页面

每个页面通常包含一个独立的子目录，其命名与页面功能对应。

## 页面归档说明

- 所有页面均为纯 HTML 文件，可离线打开或部署至任何静态服务器。
- 页面内容不依赖外部服务或特定域名，确保迁移与复用的便利性。
- 归档时保留原始文件结构与资源引用路径，避免链接失效。
- 历史版本通过 Git 提交记录保留，可随时回溯。

## 维护说明

- 欢迎提交 Issue 或 Pull Request 来改进现有页面或新增页面。
- 新增页面请遵循目录结构规范，并附上简单说明。
- 建议在提交前确保页面在主流浏览器中基本可用。
- 仓库维护者会定期审核并合并合理的贡献。

## 使用方式

1. 克隆仓库到本地：
   ```bash
   git clone https://github.com/your-username/zhc-od-resource.git
   ```
2. 直接打开 `pages/` 下任意 HTML 文件即可浏览。
3. 也可将整个仓库部署至静态托管服务，通过 URL 访问各页面。
